# v1.1.0 開発セッション (2026-04-07)

## やったこと

### 第1弾: バグ修正 + コード整理
- **clamp=64 修正**: FlipbookFrameLoader.Load() の MaxFrames=64 ハードコードを廃止。maxFrames パラメータ化し、CalculateMaxFrames(_maxSheetSize) に一本化。RunDryRunArray / RunDryRunLilToon のハードコード 64 も動的化
- **CreateFolderRecursive 統合**: ArrayBuilder / MaterialBuilder / SheetBuilder / Window の 4 箇所を FlipbookFileUtility.EnsureAssetFolderExists に統合
- **MakeReadable 統合**: ArrayBuilder / SheetBuilder の 2 箇所を FlipbookFileUtility.MakeReadable に統合
- **TryAttachModularAvatar 削除**: BuildMultiPage 内のデッドパス（未設定の MenuInstaller + 空 ObjectToggle を追加）を削除
- **未使用 import 削除**: Window.cs の using System.Linq を削除
- **GUIStyle キャッシュ化**: DrawSubInfo / DrawOutputPathPreview / DrawExtractAudioButton の new GUIStyle を static lazy init に変更

### 第2弾: UI 改善
- **タスク C: プリセット拡張**: Toolbar を _generatePrefab の上に移動し常時表示。ApplyPreset に _generatePrefab=true、MPS 時 _autoSplit=true を追加。プリセット説明 DrawSubInfo 付き。ChangeCheck スコープを Prefab/MA セクション全体に拡大
- **タスク D: FPS 計算機 Foldout 化**: PNG モードの FPS ヘルパーを Foldout で囲みデフォルト閉。_showFpsHelper SerializeField 追加
- **タスク E: 容量見積もり**: EstimateTextureBytes() ヘルパーで計算一本化（baseFrameSize=256, DXT5=1byte/pixel）。OnGUI / DryRun ダイアログ / Generate ダイアログ全 8 箇所で統一表示
- **タスク F: 秒数ガイド**: 出力モード説明 HelpBox 直後に Texture2DArray / LilToon の最大フレーム数・秒数を DrawSubInfo で表示

### タスク E 追加修正
- frameSize を _extractMaxResolution → 256（Builder の DefaultFrameSize）に修正
- RGBA32 (4 bytes/pixel) → DXT5 (1 byte/pixel) に変更
- 表示を「推定テクスチャメモリー増加: +X.XMB (PC/DXT5)」に統一

## 設計変更
- **EstimateTextureBytes の frameSize**: 当初 _extractMaxResolution を使用 → 実際の Builder は DefaultFrameSize=256 を使うため 256 に修正
- **容量表示単位**: RGBA32 未圧縮 → DXT5。VRChat パフォーマンスランク画面の「テクスチャメモリー」と揃えるため
- **プリセットスコープ**: FPS / maxSheetSize / outputFolderMode はプリセット対象外とした（ユーザーの意図的な設定を破壊するため）
