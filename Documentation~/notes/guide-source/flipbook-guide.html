Flipbook Material Generator 使い方ガイド

1. このツールでできること

Flipbook Material Generator は、動画ファイルや PNG 連番から「フリップブック」と呼ばれるパラパラ漫画式のアニメーション素材を自動生成する Unity Editor ツールです。スプライトシート（複数のフレームを 1 枚の画像にまとめたもの）・マテリアル・Prefab・EX メニュー連携まで一括で生成でき、元アバターには直接変更を加えない非破壊設計になっています。

3 つの出力モード

用途に合わせて 3 つの出力モードから選べます。

MultiPageSequence（おすすめ）は、長い動画もフレーム制限なしで扱えるモードです。音声との同期にも対応しており、EX メニューから再生 ON/OFF・ループ・リセットを制御できます。迷ったらこのモードを選んでおけば大丈夫です。

Texture2DArray は、軽量で短いループアニメに向いたモードです。フレーム数に上限がありますが、テクスチャ負荷が小さいのが特徴です。

LilToon は、lilToon シェーダーの内蔵フリップブック機能を使って再生するモードです。lilToon が導入されている環境ならシェーダーを追加する必要がなく、手軽に使えます。

各モードの詳しい違いは「3 つの出力モード解説」の章をご覧ください。

こんな人向け

  • アバターに動画や GIF 的なアニメーションを載せたい
  • 推し画像のスライドショーを作りたい
  • BGM 付きの映像演出をアバターに仕込みたい
  • フリップブックの仕組みを知らなくても、動画を指定するだけで使いたい

安心ポイント

  • 非破壊: 元アバターのデータには一切変更を加えません。生成物はすべて新しいファイルとして作られます
  • Dry Run: 生成前に「何が作られるか」をダイアログで確認できます。いきなり大量のファイルが作られる心配はありません
  • MA は optional: Modular Avatar（MA）がなくても Prefab 生成は可能です。MA が導入されていれば EX メニュー連携が自動で設定されます
  • 容量確認: 生成前に推定テクスチャメモリー増加量が表示されるので、アバターの容量への影響を事前に把握できます

2. FFmpeg セットアップ手順

動画ファイルからフリップブックを生成するには、FFmpeg というツールが必要です。この章では FFmpeg のインストール方法を説明します。

コマンドプロンプトの開き方

FFmpeg のインストールや動作確認にはコマンドプロンプトを使います。以下のいずれかの方法で開いてください。

  • Windows キーを押して「cmd」と入力 → Enter
  • Win+R → 「cmd」と入力 → Enter

インストール（winget 推奨）

コマンドプロンプトで以下のコマンドを実行します。

winget install -e --id Gyan.FFmpeg

インストールが完了したら、コマンドプロンプトを 新しく開き直して から、以下のコマンドで正しくインストールされたか確認します。

ffmpeg -version

バージョン情報が表示されれば成功です。

PATH が通らない場合の対処

ffmpeg が認識されない場合、winget のインストール先にある bin フォルダまで PATH が通っていない可能性があります。以下の手順で PATH を追加してください。

  1. Win+R → sysdm.cpl → Enter で「システムのプロパティ」を開きます
  2. 「詳細設定」タブ → 「環境変数」をクリックします
  3. ユーザー変数の Path を選択して「編集」をクリックします
  4. 「新規」をクリックし、以下のようなパスを追加します（バージョン番号は実際のものに合わせてください）
%LOCALAPPDATA%\Microsoft\WinGet\Packages\Gyan.FFmpeg_Microsoft.Winget.Source_8wekyb3d8bbwe\ffmpeg-X.X.X-full_build\bin
  1. コマンドプロンプトを開き直して ffmpeg -version で確認します

アップデート

FFmpeg を最新版に更新するには、以下のコマンドを実行します。

winget upgrade Gyan.FFmpeg

アンインストール

FFmpeg を削除するには、以下のコマンドを実行します。

winget uninstall Gyan.FFmpeg

環境変数 PATH に手動で追加していた場合は、そちらも手動で削除してください。

3. はじめての使い方（最短チュートリアル）

この章では、動画ファイルからフリップブックを生成してアバターに載せるまでの最短手順を説明します。MultiPageSequence モード（おすすめ）を使った基本的な流れです。

はじめる前に

以下が準備できていることを確認してください。

  • VCC / VPM でツールが導入済みであること
  • FFmpeg がインストール済みであること（動画入力を使う場合）

FFmpeg のインストール方法は「FFmpeg セットアップ手順」の章をご覧ください。

手順

1. ウィンドウを開く

Unity の上部メニューから Tools > Sebanne > Flipbook Material Generator を選択して、ツールウィンドウを開きます。

2. 動画ファイルを指定する

入力モード は「動画ファイル」がデフォルトで選択されています。動画ファイル のフィールドに、Assets フォルダ内にある MP4 などの動画ファイルをドラッグ＆ドロップしてください。

動画を指定すると、元動画の解像度・FPS・長さと、書き出し後の推定フレーム数が自動的に表示されます。書き出し FPS はデフォルトの 8 のままで大丈夫です（後から調整できます）。

3. 出力モードと設定を確認する

出力モード は「MultiPageSequence」がデフォルトで選択されています。Prefab / MA 設定のプリセットも おすすめ のままにしておいてください。この状態で、Prefab 生成と MA 連携がすべて ON になっています。

4. Dry Run で確認する

実行セクションにある Dry Run ボタンをクリックします。ダイアログが表示され、フレーム数・ページ構成・推定テクスチャメモリー増加量を確認できます。内容に問題がなければダイアログを閉じてください。

5. Generate で生成する

Generate ボタンをクリックすると、生成が始まります。完了するとダイアログが表示され、生成された Prefab が Project ウィンドウでハイライトされます。

6. アバターに載せる

生成された Prefab を Hierarchy ウィンドウでアバタールートの子オブジェクトとして配置します。Modular Avatar が導入されていれば、VRChat の EX メニューにフリップブックの操作項目が自動的に追加されます。あとは VRChat にアップロードして動作を確認してください。

もっと詳しく知りたい方へ

  • 音声を追加したい → 「音声つきフリップブック」の章をご覧ください
  • FPS の調整や「動画が止まって見える」問題について → 「動画の長さと FPS の目安」の章をご覧ください
  • 出力モードの詳しい違い → 「3 つの出力モード解説」の章をご覧ください

4. 3 つの出力モード解説

この章では、3 つの出力モードそれぞれの特徴・向いている用途・制約を詳しく説明します。モード選びに迷ったときの参考にしてください。

モード比較表

項目 MultiPageSequence Texture2DArray LilToon
フレーム上限 なし あり（シートサイズ依存） あり（シートサイズ依存）
音声同期 対応 不可 不可
負荷 やや重め（シート枚数分） 軽量 軽量
必要なもの （なし） （なし） lilToon シェーダー
EX メニュー 再生/ループ/リセット ON/OFF のみ ON/OFF のみ
向いてる用途 長尺・大量フレーム・音声付き 短いループアニメ lilToon 環境で手軽に

MultiPageSequence（おすすめ）

MultiPageSequence は、フレーム数の制限がなく最も多機能なモードです。PNG 連番を複数のスプライトシートに自動分割し、AnimatorController（Unity のアニメーション制御の仕組み）を使ってシームレスに再生します。

音声との同期にも対応しているため、映像と音がズレずに再生されます。音声付きのフリップブックを作りたい場合は、このモードを選んでください。

VRChat の EX メニュー（Expression Menu）からは 3 つの操作ができます。

  • Toggle: フリップブックの再生 ON/OFF を切り替えます
  • Loop: ループ再生の ON/OFF を切り替えます（デフォルトは ON）
  • Reset: 先頭から再生し直します

テクスチャはページ数分（スプライトシートの枚数分）読み込まれるため、他の 2 モードと比べるとやや負荷が大きくなります。ただし、フレーム数が同じならどのモードでも総テクスチャ量はほぼ同じです。

Texture2DArray

Texture2DArray は、Texture2DArray（複数のテクスチャを配列として 1 つにまとめた形式）とカスタムシェーダーでフリップブック再生するモードです。

テクスチャ 1 つで完結するため軽量ですが、フレーム数に上限があります。上限は最大シートサイズの設定によって変わり、ツールの UI に「最大 N フレーム ≈ X.X 秒」という形でガイドが表示されます。

音声との同期はできません。音声を追加すること自体は可能ですが、BGM や環境音など、映像とのタイミング合わせが不要な用途に限られます。

ループ再生専用のモードで、EX メニューからは ON/OFF の切り替えのみ行えます。

LilToon

LilToon は、lilToon シェーダーに内蔵されているフリップブック再生機能（Main2nd DecalAnimation）を使うモードです。

lilToon が導入されている環境なら追加のシェーダーが不要で、手軽にフリップブックを組み込めます。すでに lilToon を使っているアバターに載せたい場合に便利です。

Texture2DArray と同様にフレーム数に上限があり、音声との同期もできません。ループ再生専用で、EX メニューからは ON/OFF の切り替えのみです。

lilToon がプロジェクトに導入されていない場合、出力モード選択時に警告が表示されます。その場合は Texture2DArray モードをお使いください。

Quad の向きについて（v1.1.0 時点の制約）

生成される Quad（フリップブックを表示する板）は、Unity のデフォルトの法線方向（+Z 方向）で作られるため、Scene View のカメラ方向によっては裏面が見えているように感じることがあります。

ただし、描画自体には問題ありません。すべてのモードで両面描画が有効になっており、どの方向から見ても正しく表示されます。Scene View での見え方が気になる場合は、Play モードに入って Game View から確認してください。

今後のアップデートで、Scene View 上にガイド表示を追加して分かりやすくする予定です。

どのモードを選べばいい？

迷ったら MultiPageSequence を選んでください。フレーム制限なし・音声同期対応・EX メニュー操作と、一番機能が充実しています。

5 秒以内の短いループアニメなら Texture2DArray が軽量でおすすめです。lilToon を使っているアバターでシェーダーを増やしたくない場合は LilToon が手軽です。

5. 入力設定

この章では、入力設定セクションの使い方を説明します。素材となる動画ファイルまたは PNG 連番フォルダの指定と、再生 FPS の設定を行います。

入力モードの選択

入力モード で「動画ファイル」か「PNG連番フォルダ」を選択します。

動画ファイルを直接指定する方法が手軽でおすすめです。この場合は FFmpeg が必要になります。PNG 連番は、事前に自分で動画から PNG を書き出している場合や、画像編集ソフトで作ったフレームを使いたい場合に選びます。

動画ファイル入力

動画ファイルの指定

動画ファイル のフィールドに、Assets フォルダ内にある動画ファイルをドラッグ＆ドロップで指定します。対応している形式は mp4、webm、mov、avi、mkv です。

動画を指定すると、元動画の解像度・FPS・長さと推定フレーム数が自動的に表示されます。

FFmpeg がインストールされていない場合や PATH が通っていない場合は、警告メッセージが表示されます。FFmpeg のセットアップについては「FFmpeg セットアップ手順」の章をご覧ください。

書き出し FPS

書き出し FPS は、動画を PNG フレームに変換する際の FPS（1 秒あたりのフレーム数）を指定します。これは元動画の FPS とは別の値です。

FPS を低くするとフレーム数が減り、テクスチャ容量が軽くなります。デフォルトは 8fps で、多くの場合これで十分です。FPS の考え方について詳しくは「動画の長さと FPS の目安」の章をご覧ください。

トリミング

トリミング にチェックを入れると、動画の一部分だけを使えます。開始（秒） で使い始める位置を、長さ（秒） で使用する区間の長さを指定します。

開始＋長さが動画全体の長さを超えている場合は警告が表示されます。トリミングを設定すると、トリミング後の推定フレーム数が自動的に表示されます。

PNG 連番フォルダ入力

入力フォルダの指定

入力フォルダ のフィールドに、PNG 連番が入った Assets 内のフォルダを指定します。フォルダ内の PNG ファイルがファイル名順に自動的にフレームとして読み込まれます。

PNG連番 FPS

PNG連番 FPS には、PNG を書き出した時の FPS を入力します。元動画の FPS ではなく、FFmpeg 等で PNG に変換した際に指定した FPS を入力してください。

書き出し時の FPS が分からない場合は、FPS 計算機を使って逆算できます。

FPS 計算機

FPS 計算機（デフォルトで閉じている折りたたみ）を開くと、PNG の枚数と元動画の長さから FPS を逆算できます。

  1. PNG 枚数 に PNG ファイルの数を入力します。入力フォルダから取得 ボタンで自動カウントもできます
  2. 動画 分 と 動画 秒 に元動画の長さを入力します
  3. 計算結果が自動表示されるので、FPS に適用 ボタンで PNG連番 FPS に反映できます

6. 動画の長さと FPS の目安

この章では、書き出し FPS の考え方と、動画の長さとの関係を説明します。「動画が止まって見える」「テクスチャ容量が大きすぎる」といった問題の多くは、FPS の調整で解決できます。

大切なポイント

動画が長い ＝ ダメ、ではありません。 テクスチャ容量に影響するのは「総フレーム数」であり、総フレーム数は「動画の秒数 × 書き出し FPS」で決まります。FPS を下げればフレーム数を抑えられるので、長い動画でも扱えます。

推奨フレーム数の目安

総フレーム数は 200〜300 枚以内 を目標にすると、容量と見た目のバランスが取りやすいです。

ツールの FPS 計算機 を使えば、「目標フレーム数 ÷ 動画秒数 ＝ 推奨 FPS」を逆算できます。たとえば 30 秒の動画で 240 枚に収めたい場合、240 ÷ 30 ＝ 8fps が目安になります。

「止まって見える」原因

フリップブックが動いているはずなのに止まって見える場合、FPS が高すぎることが原因であることがほとんどです。

FPS が高いと総フレーム数が膨大になり、1 枚のスプライトシートに詰め込まれるフレーム同士の変化量が非常に小さくなります。結果として、アニメーションが動いているのに見た目にはほぼ変化がない状態になります。

対処法

書き出し FPS を下げてください。 たとえば 30 秒の動画であれば、30fps → 5〜8fps に下げると 150〜240 枚程度に収まり、フレーム間の変化が目に見えるようになります。

デフォルトの 8fps は多くの場合に適切な値です。迷ったらまず 8fps で Dry Run してみて、フレーム数と推定テクスチャメモリーを確認するのがおすすめです。

テスト素材について

動作確認には、フレーム間の変化が目で見て分かりやすい素材を使うと確実です。数字が順番に表示される動画や、色がゆっくり変化する動画などが適しています。実写のように変化が微妙な素材だと、FPS の影響が確認しにくい場合があります。

7. 出力設定

この章では、出力設定セクションの使い方を説明します。出力モードの選択と、生成ファイルの保存先・管理方法を設定します。

出力モード

出力モード で Texture2DArray / LilToon / MultiPageSequence のいずれかを選択します。選択中のモードに応じた簡単な説明がウィンドウに表示されます。

Texture2DArray と LilToon では、現在の FPS 設定に基づいた「最大 N フレーム ≈ X.X 秒」という秒数ガイドも表示されるので、どのくらいの長さの動画を扱えるかの目安にできます。

各モードの詳しい違いについては「3 つの出力モード解説」の章をご覧ください。

出力先

出力先 では、生成したファイルをどこに保存するかを 3 つの選択肢から選べます。

元ソース直下 を選ぶと、入力フォルダの直下に Generated_Flipbook/ フォルダが作成され、そこに生成物が保存されます。素材と生成物を近くに置きたい場合に便利です。

ツール共通フォルダ（デフォルト）を選ぶと、Assets/Sebanne/FlipbookMaterialGenerator/Generated_Flipbook/ に保存されます。素材と生成物を分けて管理したい場合におすすめです。

フォルダを指定 を選ぶと、出力フォルダ のフィールドが表示され、任意のフォルダを指定できます。

いずれの場合も、現在の出力先 に実際の保存パスが表示されるので、生成前に確認できます。

出力名とスロット

出力名 はフォルダ名の一部になります。動画やフォルダを指定すると自動でファイル名が入りますが、手動で自由に変更することもできます。

スロット は生成物の保存先フォルダの管理単位です。Generated_Flipbook/01_名前/ のように番号付きフォルダで管理されます。

自動（新規作成） を選ぶと、次の番号で新しいフォルダが作られます。既存のスロットを選ぶと、そのフォルダの中身が上書きされます（上書き前に警告が表示されるので安心してください）。

同じ素材で設定を変えて何度か生成したい場合は、スロットを分けておくと比較しやすくなります。

スロットブラウザ

生成済みスロット一覧 を開くと、過去に生成したスロットの一覧を確認できます。各スロットにはシート数・マテリアル数・Prefab の有無・Audio の有無が表示されます。

スロット名をクリックすると、スロットドロップダウンでそのスロットが選択状態になります。フォルダを開く ボタンで Project ウィンドウにフォルダをハイライト表示できます。

不要になったスロットは 削除 ボタンで一括削除できます。確認ダイアログが表示された後、スロット内のファイルが完全削除されます（ゴミ箱には送られません）。

8. Prefab / MA 設定

この章では、Prefab 生成と Modular Avatar（MA）連携の設定を説明します。プリセット機能を使えば、多くの場合デフォルトのままで適切な設定になります。

プリセット

セクション上部にある おすすめ / カスタム の切り替えで、設定をまとめて管理できます。

おすすめ を選ぶと、Prefab 生成 ON・MA 連携すべて ON・AudioSource OFF の状態に設定されます（MultiPageSequence モードでは自動分割 ON も含まれます）。基本はこのまま使えば問題ありません。

カスタム を選ぶと、各設定を自由に変更できます。いずれかの設定を手動で変えると、自動で「カスタム」に切り替わります。

Prefab 生成

Prefab も生成する を ON にすると、Quad（フリップブックを表示する板）とマテリアルを含む Prefab が生成されます。アバターの子オブジェクトに配置するだけで使えるので、通常は ON のままにしてください。

OFF にすると、マテリアルとテクスチャだけが生成されます。自分で Quad や表示先を用意したい場合向けです。

MA 連携（MultiPageSequence の場合）

MultiPageSequence モードでは、以下の MA 設定が表示されます。

MA Merge Animator は、生成された AnimatorController をアバターの FX レイヤー（VRChat がアバターの表情やギミックを制御する仕組み）に統合する設定です。VRChat アバターで使う場合は ON にしてください。OFF にすると、AnimatorController が FX に統合されないため EX メニューからの操作ができなくなります。

MA Object Toggle を ON にすると、VRChat の EX メニューからフリップブックの ON/OFF を切り替えられるようになります。

トグル名 は EX メニューに表示される名前です。デフォルトは「Flipbook」ですが、自由に変更できます。入力フォルダから取得 ボタンで入力フォルダの名前を取得することもできます。

音源を追加 を ON にすると、音声付きのフリップブックを生成できます。詳しくは「音声つきフリップブック」の章をご覧ください。

MultiPageSequence モードの EX メニューには 3 つの操作項目が追加されます。

  • Toggle: フリップブックの再生 ON/OFF（デフォルト OFF）
  • Loop: ループ再生の ON/OFF（デフォルト ON）
  • Reset: 先頭から再生し直し（ボタン型）

MA 連携（Texture2DArray / LilToon の場合）

Texture2DArray と LilToon モードでは、設定項目がシンプルになります。

MA Object Toggle を追加 を ON にすると、EX メニューからフリップブックの ON/OFF を切り替えられます。MA Menu を追加 を ON にすると、EX メニューに項目が追加されます。

トグル名 と 音源を追加 は MultiPageSequence と同じ使い方です。

これらのモードには AnimatorController がないため、EX メニューは Toggle（ON/OFF）のみで、Loop や Reset はありません。

MA がなくても使える

Modular Avatar が導入されていなくても、Prefab 生成は問題なく行えます。EX メニュー連携が自動設定されないだけで、Hierarchy 上でフリップブックの GameObject を手動で ON/OFF すれば動作します。

MA は VCC から導入できます。EX メニューからの操作が必要な場合は、MA の導入をおすすめします。

9. 音声つきフリップブック

この章では、フリップブックに音声を追加する方法を説明します。映像と音声を同期させたい場合のモード選びについても解説します。

音声を追加するには

Prefab / MA 設定セクションで 音源を追加 を ON にして、AudioClip のフィールドに WAV ファイル等をドラッグ＆ドロップで指定します。

動画ファイルを入力に使っている場合は、動画から音声を抽出 ボタンを使うと、動画から音声を自動で取り出して設定できます。自分で WAV を用意する手間が省けます。

音声同期の違い

音声と映像の同期ができるかどうかは、出力モードによって異なります。

モード 音声同期 音声の使い方
MultiPageSequence 対応 映像と音声がズレずに再生されます
Texture2DArray 不可 BGM・環境音など同期不要の用途に使えます
LilToon 不可 BGM・環境音など同期不要の用途に使えます

MultiPageSequence は AnimatorController（Unity のアニメーション制御の仕組み）で映像を再生しているため、音声との同期が可能です。映像と音を合わせたい場合はこのモードを選んでください。

Texture2DArray と LilToon はシェーダー内部の時間で再生するため、AnimatorController とは連動できません。音声を追加すること自体は可能ですが、映像との同期は保証されません。ループ BGM や環境音など、タイミングを合わせる必要がない音声を添えたい場合に使えます。

音声抽出機能

動画から音声を抽出 ボタンは、入力モードが「動画ファイル」の場合に表示されます。

このボタンをクリックすると、FFmpeg を使って動画ファイルから音声が WAV 形式で抽出されます。抽出された WAV はスロット内の Audio/ フォルダに保存され、AudioClip に自動設定されます。

トリミング設定がある場合は、音声にも同じトリミングが適用されます（「トリミング設定が音声にも適用されます」というメッセージが表示されます）。

音声トラックがない動画の場合は「この動画に音声トラックはありません。」という警告が表示されます。

音声追加時の注意

音源を追加 が ON の状態で AudioClip が未指定のまま Generate すると、音声なしで生成されます。実行セクションに「AudioClip が指定されていません」という警告が表示されるので、意図的に音声なしにしたい場合以外は AudioClip を設定してください。

10. 上級設定

この章では、上級設定セクションの各項目を説明します。通常はデフォルトのまま変更する必要はありません。テクスチャ容量の微調整やトラブルシューティング時に参照してください。

上級設定の開き方

ウィンドウ最下部にある 上級設定 の折りたたみをクリックして展開します。デフォルトでは閉じた状態です。

最大解像度（動画入力のみ）

最大解像度 は、FFmpeg で動画を PNG に書き出す際の最大ピクセルサイズです。256 / 512 / 1024 から選択でき、デフォルトは 512 です。

元動画の解像度がこの値より大きい場合、書き出し時に縮小されます。値を下げるとテクスチャ容量が軽くなりますが、画質は落ちます。VRChat のアバターに載せる用途であれば 512 で十分なことがほとんどです。

この項目は動画入力モードの場合のみ表示されます。PNG 連番入力の場合は、元の PNG の解像度がそのまま使われます。

最大シートサイズ

最大シートサイズ は、1 枚のスプライトシート（複数フレームを 1 枚にまとめた画像）の最大ピクセルサイズです。512 / 1024 / 2048 / 4096 から選択でき、デフォルトは 2048 です。

通常は変える必要はありません。 シートサイズを変えても、総テクスチャ容量はほぼ変わりません。1 枚のシートが大きくなるか、シートの枚数が増えるかの違いだけです。テクスチャ容量に一番影響するのは FPS とフレーム数です。

なお、この設定は Unity の Import Settings にある「Max Size」とは別のものです。ツールの最大シートサイズは「生成する画像のピクセルサイズの上限」で、Unity の Max Size は「ビルド時に圧縮する上限」です。

自動分割（MultiPageSequence のみ）

自動分割 は、MultiPageSequence モードで PNG 連番を何フレームごとにページ分割するかを制御します。デフォルトで ON になっています。

ON の場合、最大シートサイズから最適な 1 ページあたりのフレーム数が自動計算されます。計算結果は 1ページ最大フレーム数 の欄と、その下の HelpBox に表示されます。

手動で調整することも可能ですが、通常は自動のままで問題ありません。

コンソールログ

コンソールにログを出力 を ON にすると、生成過程の詳細なログが Unity の Console ウィンドウに出力されます。デフォルトは OFF です。

何かうまくいかない場合のトラブルシューティングに役立ちます。通常の使用では OFF のままで大丈夫です。

11. FAQ・トラブルシューティング

この章では、よくある質問と問題の解決方法をまとめています。

動画が止まって見える / カクつく

FPS が高すぎてフレーム数が膨大になっている可能性があります。書き出し FPS を下げてみてください。5〜8fps がおすすめです。

「動画が長い ＝ ダメ」ではありません。FPS を下げればフレーム数を抑えられるので、長い動画でも扱えます。詳しくは「動画の長さと FPS の目安」の章をご覧ください。

テクスチャメモリーが大きすぎる

テクスチャ容量を減らす方法はいくつかあります。

  • 書き出し FPS を下げる: フレーム数が減るので、最も効果的です
  • トリミングで使用区間を短くする: 動画全体ではなく一部だけ使えば、その分フレーム数が減ります
  • 最大解像度を下げる: 上級設定の 最大解像度 を 512 → 256 に変えると容量が減ります（画質は落ちます）

実行セクションに「推定テクスチャメモリー増加」が表示されるので、Generate 前に確認してください。

「FFmpeg がパスに見つかりません」と表示される

FFmpeg がインストールされていないか、PATH が通っていない状態です。コマンドプロンプトで ffmpeg -version と入力して動作するか確認してください。セットアップ手順は「FFmpeg セットアップ手順」の章をご覧ください。

「フレーム数が収容上限を超えています」と表示される

Texture2DArray と LilToon モードにはフレーム数の上限があります。以下のいずれかで対処してください。

  1. トリミング で使用区間を短くする
  2. 書き出し FPS を下げてフレーム数を減らす
  3. 出力モード を MultiPageSequence に切り替える（フレーム上限なし）

「lilToon がプロジェクトに導入されていません」と表示される

LilToon モードは lilToon シェーダーが必要です。VCC から lilToon を導入するか、Texture2DArray モードを使ってください。

EX メニューに表示されない

以下の点を確認してください。

  1. Modular Avatar（MA）がプロジェクトに導入されているか
  2. Prefab も生成する が ON になっているか
  3. MultiPageSequence モードの場合: MA Merge Animator と MA Object Toggle が ON になっているか
  4. Texture2DArray / LilToon モードの場合: MA Object Toggle を追加 が ON になっているか
  5. 生成した Prefab がアバタールートの子オブジェクトに配置されているか

MA が導入されていない場合は、EX メニュー連携は使えません。Hierarchy 上で手動で ON/OFF する運用になります。

生成したものを削除したい

出力設定セクションの 生成済みスロット一覧 を開いて、削除したいスロットの 削除 ボタンをクリックしてください。確認ダイアログが出た後、スロット内のファイルが完全に削除されます（ゴミ箱には送られません）。

Quad が裏向きに見える

Scene View のカメラ角度による見え方の問題です。実際の描画はすべてのモードで両面対応済みなので、正常に表示されます。

見え方が気になる場合は、Play モードに入って Game View から確認してください。
